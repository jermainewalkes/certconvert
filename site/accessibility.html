Accessibility

Accessibility is a launch requirement for CertConvert, not an afterthought.

The app

CertConvert is built so it can be used comfortably with assistive technology:

  • Full keyboard access. Every control is reachable and operable by keyboard — you do not need a mouse to inspect, convert, chain, work with keys or generate certificates.
  • Screen-reader labels. Icon-only and otherwise ambiguous controls carry text labels so a screen reader announces what they do.
  • Live status announcements. The result of an operation — a successful conversion, a chain that validates or fails, a key that does or doesn’t match — is announced through a live region rather than shown only visually.

This website

The site uses semantic HTML, keeps a readable text size and colour contrast, respects your system light or dark theme, and works without JavaScript for reading content. If any page is awkward to use, please tell us.

Found a barrier?

Accessibility problems are treated as bugs. If anything in the app or on this site is awkward with a keyboard, a screen reader or any other assistive technology, please raise it — open an issue on the GitHub issue tracker or get in touch, and it will be looked at.
