Documentation

A quick tour of the app, the command-line verbs with worked examples, and answers to the common snags. Run certconvert --help for the full CLI list.

Using the app

CertConvert opens on a sidebar with five views. Most jobs are drag-and-drop: point a view at a file and it detects the format from the file’s content, not its extension.

Inspect

Open or drag in any certificate, key, CSR or bundle. CertConvert decodes it and shows the subject, issuer, validity, serial, public-key algorithm, SANs, key usage and SHA-256 / SHA-1 fingerprints. Use Copy PEM or Copy SHA-256 to grab either. If the file is a password-protected PFX or key, type the password and choose Unlock.

Convert

Load a file, pick the target format — PEM, DER (.cer), PKCS #7 (.p7b) or PKCS #12 (.pfx) — and save. To build a PFX you can supply the matching private key and set an export password.

Chain

Add your root, intermediate and device certificates in any order. CertConvert arranges them into a valid chain, validates it offline and reports pass or fail with a reason. Export the assembled chain as a PEM bundle, a P7B, or a PFX with the key attached.

Keys

Convert a private key between PKCS #8, PKCS #1 and SEC 1 formats, add or remove a passphrase, or check whether a key matches a certificate — CertConvert compares their public keys and tells you yes or no.

Generate

Create a new RSA or ECDSA key, a certificate signing request, or a self-signed certificate. Fill in the subject fields, add SANs (DNS names and IP addresses) and set CA options and a validity period, then export in whatever format you need.

Command-line reference

The same executable is a CLI when given arguments and a GUI when not. A few worked examples:

certconvert inspect device.pem
certconvert convert device.pem -o device.cer            # PEM → DER
certconvert convert bundle.p7b -o bundle.pem            # PKCS#7 → PEM
certconvert chain build device.pem ca.pem root.pem -o chain.pfx \
            --key device.key --out-password secret
certconvert chain verify chain.p7b
certconvert key convert device.key -o device_pkcs8.key --to pkcs8
certconvert key match --cert device.pem --key device.key
certconvert gen selfsigned --new-key p256 --key-out dev.key \
            --cn device.local --dns device.local -o dev.pem
certconvert update                                      # check GitHub for a newer version
certconvert update --install                            # download, verify and apply it

Exit codes: 0 success, 1 usage error, 2 failure (including an invalid chain or a key that does not match). That makes CertConvert easy to drop into a build pipeline or a shell script.

Troubleshooting

macOS: “CertConvert can’t be opened”

The build is unsigned, so Gatekeeper blocks the first open. Right-click the app and choose Open (one time), or allow it under System Settings → Privacy & Security.

Windows: “Windows protected your PC”

SmartScreen on an unsigned binary. Choose More info → Run anyway (one time).

“…is password-protected”

The file is an encrypted PFX or key. Type the password in the password field and use Unlock in the app, or --password on the CLI. Errors always name the file that needs it.

Windows CLI output interleaves with the prompt

The executable is a GUI program attaching to your console, so output can arrive after the prompt returns. Press Enter to get the prompt back.

Which build am I running?

Run certconvert --version; it prints the version plus the exact git commit it was built from.

Update didn’t apply, or partially applied

Download the latest zip from Releases and replace the app manually. If a self-update was interrupted, a recovery copy of the previous version is kept beside the app (CertConvert.app.bak on macOS, CertConvert.exe.old on Windows) until the next launch.

“Check For Updates” can’t reach GitHub

That check needs internet; on an offline machine it simply reports the failure and changes nothing. Everything else works offline.
