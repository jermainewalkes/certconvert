Downloads are hosted on GitHub Releases, where each zip is listed with its SHA-256 checksum. Older versions are available there too.

Per-platform downloads

Grab the zip that matches your machine from the latest release (replace <version> with the version above):

PlatformFile
macOS (Apple Silicon)CertConvert-<version>-osx-arm64.zip
macOS (Intel)CertConvert-<version>-osx-x64.zip
Windows x64CertConvert-<version>-win-x64.zip

Unzip it. On macOS, move CertConvert.app to Applications; on Windows, run CertConvert.exe. Each release lists the SHA-256 of every zip so you can verify your download.

First launch on an unsigned build

Heads up: current builds are not yet code-signed, so both operating systems will warn you the first time you open the app. This is expected — here is the one-time step for each.

macOS — “CertConvert can’t be opened”

Because the build is unsigned, Gatekeeper blocks the first open. Right-click (or Control-click) CertConvert.app and choose Open, then confirm — you only need to do this once. Alternatively, allow it under System Settings → Privacy & Security after the first attempt.

Windows — “Windows protected your PC”

SmartScreen shows this for any unsigned binary. Click More info → Run anyway — once — to launch CertConvert.exe.

Build from source

Prefer to build it yourself? You need the .NET 10 SDK.

git clone https://github.com/jermainewalkes/certconvert.git
cd certconvert
dotnet run --project src/CertConvert    # launches the GUI

Self-contained release builds for all three targets (osx-x64, osx-arm64, win-x64) are produced by build/publish.sh. See the repository README for the full development notes.

System requirements

  • macOS: a recent macOS on Intel or Apple Silicon. The build is self-contained — no separate .NET install.
  • Windows: 64-bit Windows 10 or 11. Self-contained — no separate .NET install.
  • Network: none required. The app runs fully offline; the only optional network use is a manual check for updates on GitHub.
